# Screenshots for the README

Drop the following PNG (or JPG) files here — they are referenced by the main
`README.md`. Until they exist, the Screenshots section will show broken images,
so add them before pushing.

| File | What to capture |
| --- | --- |
| `dashboard.png` | The main dashboard with the left sidebar and the **Today** view (a few tasks, ideally with priorities/labels visible). |
| `task-editor.png` | The full task editor open, showing the **chip row** (date, priority, labels, recurrence, deadline, reminder). |
| `quick-add.png` | The **quick-add** modal with a natural-language example typed in (e.g. `Write report tomorrow p1 #work`). |
| `reminders.png` | The **reminders** popover open (relative presets + the “Date & time” option). |

Tips:
- Use a clean theme and a couple of realistic (non-private) example tasks.
- Capture at a comfortable width; GitHub scales images down to the content width.
- Keep filenames exactly as above (they are hard-coded in `README.md`).

Optional extras you could add later (and reference in `README.md`):
`theming.png` (Style Settings / color variants), `subtasks.png` (nested tasks with
connector lines), `calendar.png` once the calendar view ships.
